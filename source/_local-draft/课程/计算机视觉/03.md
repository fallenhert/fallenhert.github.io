## Harris algorithm

**Rules for finding the suitable feature patch window**

- Two Eigen values $(\lambda_{max}, \lambda_{min})$ exist for Harris Matrix A

- $\lambda_{min}$ must be big enough
- $\lambda_{max} \approx \lambda_{min}$ is a good criterion for corner features.(both $\lambda_{max}$, $\lambda_{min}$ should be large.)

**Harris Algorithm**

扫描图片中的所有像素点，框出每一个像素点周围的NxN的相邻像素区域作为矩阵。

计算每一个矩阵的特征值

筛选出特征值最相近且最大的矩阵

step1:

- scan all x,y in the image
- for each pixel (x,y) find the neighborhood N(x,y) to form a search window_N

step2:

- inside search window_N centered at (x,y), Find Harris Matrix(A)

- Find Eigen 2 values of A(x,y) = $\lambda_{max}, \lambda_{min}$

> **A more efficient approach(Quick algorithm using R)**
>
> Faster method: no need to find Eigen values
>
> use function $R = det(A) - k * trace(A)^{2}$, k = 0.04 - 0.06 by trial-and-error


step3:(after all $\lambda(x,y)$ are found:)

- sort all $\lambda_{min}(x,y)$, discard pixels with small $\lambda_{min}$
- discard pixels with large $\lambda_{large} - \lambda_{min}$
- remaining features are corner features
